首頁：企業導入AI之戰略架構與實施路徑
簡介：AI 與 程式系統 定義
- AI 是將大數據進行分析、預測、決策的系統
- 程式系統是將資料進行收集、整理、分析的系統

一、企業專屬資料庫建立（AI 檢索來源）
針對企業內各部門建置專屬資料庫，作為後續 AI 檢索與自動化生成之基礎：
1. 客戶資料區塊：歷來郵件、會議紀錄、客抱訂單、客戶企業營運、客戶成員資料庫
2. 市場趨勢資料區塊：各事業群市場歷史、龍頭財報、各大論壇新聞報導、展覽廠商資料庫
3. 產品資料區塊：Landscape全產品、評論大數據、網路評測、歷史產品資料庫
4. 研發資料區塊：CPC規格明細、客抱訂單、創新產品/提案、安規測試資料庫
5. 技術資料區塊：專利申請、專利競爭者、電機電控、設備能力評估資料庫
6. 成本資料區塊：TIP TOP、COGS、估價成本分析資料庫、估價AI助手
7. 視傳資料區塊：客戶說明書、Guidebook、客戶色系/字型/Logo、展覽設計/型錄蒐集、樣品/競品攝影紀錄資料庫

二、行銷部企劃案分成
1. OEM：由客戶會議訂出規格與需求，由行銷部統籌資料後編輯企劃案，搭配銷售被動完成
2. ODM：針對客戶進行產品規格研究，提出Sales Kit 主動提案，由行銷部統籌規劃後撰寫企劃案
3. OBM：進行市場缺口、趨勢研究，由行銷部統籌規劃後撰寫企劃案，提出Sales Kit 主動完成

三、針對 OEM 的流程分析與 AI/程式導入
中央上方為主要流程。導入後總工時大幅縮短：
1. 客需表 (工時：16h → 1h)
   - 協作：人工 10%、內部模型 90%
   - 產出：AI 產出 Json 資料
   - 程式處理：讀取 Json 資料並自動寫入公版表單
2. 企劃案背景簡介 (工時：24h → 1h)
   - 檢索庫：[客戶]、[市場趨勢]、[產品]
   - 協作：內部模型 30%、外部模型 60%
   - 產出：AI 產出評估報告 (市場、客戶、競爭樣態)
   - 程式處理：讀取 Json 資料寫入模組化版型
3. 產品競品比較表 (工時：8h → 1h)
   - 檢索庫：[產品]
   - 協作：外部模型 100%
   - 產出：AI 產出 Json 資料
   - 程式處理：讀取 Json 資料寫入模組化版型
4. 產品規格明細表 (工時：8h → 3h)
   - 檢索庫：[研發]
   - 協作：人工 30%、內部模型助手 70%
   - 產出：AI 產出規格與安規建議
   - 程式處理：人工根據建議寫入公版表單
5. 成本估價明細 (工時：80h → 70h)
   - 檢索庫：[成本]
   - 協作：人工 90%、內部模型助手 10%
   - 產出：AI 提供估價參考建議
   - 程式處理：無，寫入公版表單
6. 材料成本和模具費用 (工時：2h → 0.2h) & 效益分析表 (工時：60h → 6.4h)
   - 協作：人工 10%、程式 90%
   - 程式處理：讀取 Excel 或前面的 Json 資料寫入公版表單

四、針對現階段 ODM 的流程分析與 AI/程式導入
前端研究與企劃流程 (總工時由 220h 縮減為 38.4h)：
1. 客戶產品調查 (工時：120h → 16h)
   - 檢索庫：[客戶]、[市場趨勢]、[產品]
   - 協作：內部模型 10%、外部模型 90%
   - 產出：AI 評估報告 (評論、缺口、歷史、客戶動向)
2. 設計發想 (工時：40h → 16h)
   - 檢索庫：[產品]、[研發]、[技術]
   - 協作：人工 50%、內部模型助手 50%
   - 產出：AI 評估設計可行性
3. 產品競品比較表、客需表、企劃案背景簡介 (工時：48h → 3h)
   - 與 OEM 邏輯類似，多依賴外部與內部 AI 模型產出 Json 資料與評估報告，再透過程式讀取寫入模組化版型或公版表單。
4. 產品規格明細表、成本估價明細、材料與效益分析 (工時：92h → 73.4h)
   - 同樣整合內部模型提供規格與估價建議，人工審核寫入，程式進行 Excel 與成本資料整併寫入公版表單。
